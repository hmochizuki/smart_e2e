# smart_e2e アーキテクチャ概要

## 目的

E2Eテストの「録画 → 保存 → 実行 → 失敗時にLLMが自動修復」までを1つのTauriアプリで完結させる、保守可能なE2E基盤ツール。

## 全体ブロック図

```
+---------------------------------------------------------------+
|                       Tauri デスクトップアプリ                 |
|                                                               |
|  +----------------------+        +-------------------------+  |
|  |  React Frontend      |  IPC   |   TS Backend (本体)     |  |
|  |  (Vite + React)      | <----> |   Tauri command 層       |  |
|  |  - suite一覧/編集    |        |   - 入出力スキーマ検証   |  |
|  |  - 実行ビューア      |        |   - runner / persistence |  |
|  |  - 修復diff          |        |     のオーケストレーション|  |
|  +----------------------+        +-----------+-------------+  |
|                                              |                |
|                              +---------------+----------+     |
|                              |                          |     |
|                       +------+-------+         +--------+--+  |
|                       |   runner     |         | persistence| |
|                       |  (Playwright |         |  (SQLite + | |
|                       |   + LLM修復) |         |   Drizzle) | |
|                       +------+-------+         +--------+--+  |
|                              |                          |    |
|                              v                          v    |
|                       Playwright (Docker隔離)     SQLite file |
+---------------------------------------------------------------+

                       ^
                       | Rust の役目は薄い shell:
                       | - Tauri runtime
                       | - OS連携 (ファイル/シェル)
                       | 書き込みロジックは原則すべて TS 側
```

## パッケージ責務

| パッケージ               | 役目                                                                                                                                         |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `@smart-e2e/shared`      | zod による型/スキーマ定義。全パッケージで共有する純粋ドメインのみ。副作用なし。                                                              |
| `@smart-e2e/runner`      | Playwright によるテスト実行、4分類エラー判定 (要素未検出/タイミング/アサーション失敗/ネットワーク/その他)、LLM呼び出しによる自動修復ループ。 |
| `@smart-e2e/persistence` | SQLite + Drizzle ORM。suite/run/repair などの永続化。Drizzle スキーマと `shared` の zod スキーマの整合性は専用テストで担保。                 |
| `@smart-e2e/app`         | Tauri アプリ本体 (TS BE + React frontend)。Tauri command 層が `runner` / `persistence` を呼び出すゲートウェイ。                              |

## データフロー

1. フロントエンド (React) からユーザー操作 → `invoke('command-name', payload)` で Tauri command 呼び出し
2. Tauri command (TS) が zod で payload を検証
3. 必要に応じて `runner` を起動し、Playwright スクリプトを実行
4. 実行ログ/結果は `persistence` を介して SQLite に永続化
5. フロントに stream/結果を返却し、ビューアで可視化

## フルオート修復ループ概略

```
test実行
  └── 失敗
        └── エラー分類 (4種類)
              └── 修復可能なら LLM に context (DOM diff, セレクタ候補, 直近の操作) を投げる
                    └── 提案パッチを suite に適用
                          └── リトライ (最大N回)
                                └── 成功なら confirmed として保存 / 失敗ならユーザーへ報告
```

## 設計指針

- **書き込み一本化**: 永続化への書き込みは `persistence` のリポジトリ層のみ。Rust 側からは書かない。
- **薄い Rust shell**: Tauri 側のRustは window生成と OS API ブリッジに留め、ビジネスロジックは TS。
- **型はzodで一意化**: `shared` の zod スキーマを単一の真実とし、Drizzle スキーマ側との整合性はテストで強制。
- **静的解析の二段構え**: oxlint で広域 (速い)、ESLint は type-aware 限定 (遅いが必要)。重複排除。
